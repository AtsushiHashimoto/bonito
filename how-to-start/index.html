セットアップのガイド(ユーザ向け)

前提

  1. サーバに既にbonitoが設定されていること
  2. ユーザがdockerコマンドを実行できること(参考)

初回設定

  1. 作成するimageの初期設定をするためのDockerfileを作成します．
    完全なDockerfileはbonitoが自動で生成するのでユーザは必要なモジュールのインストールコマンドのみを，Dockerfile_tailファイルに記述します．
    % mkdir ~/.bonito

    % vim ~/.bonito/default/Dockerfile_tail
    記入例) apt install vim pip install torch torchvision
    pip install tensorflow

  2. bonitoコマンドでimageを作成します．
    % bonito create

Docker Containerの起動と接続

ユーザは下記コマンドを実行するだけで，自分専用の仮想PCに接続することができます．

% bonito run

特にjupyterを利用する場合には，bonito create後，あるいはbonito shutdown後の初回のbonito run実行時に下記のようにオプションを指定することで，ホストPC上の任意のポート(下記の例では18888)をコンテナ内のjupyter用の8888ポートに接続することができます．

%bonito run -o '-p 18888:8888'

docker ユーザ向けの解説

bonitoでは，簡単のため，docker runとdocker attachの区別をせず，bonito runコマンドのみを用います．bonito runを実行すると，未だコンテナがなければdocker runを既にコンテナが存在する場合にはdocker attachを，自動的に判定して実行します．

また，-oオプションは，docker runコマンドに直接オプションを引き渡すためのものです．つまり，-o '-p 18888:8888' とすると，docker run ...(bonitoが作成する様々なオプション)... -p 18888:8888が実行されます．なお，通常の設定では，bonitoコマンドの実行時に，bonitoが生成したdockerコマンドは標準出力にプリントされます．

Dockerコンテナの削除

%bonito shutdown

dockerユーザ向けの解説

bonitoは下記のルールでimageおよびコンテナの名前を自動生成します．

  • [image] bonito/ユーザ名/プロジェクト名:latest
  • [container] bonito.ユーザ名.プロジェクト名

なお，「ユーザ名」はbonitoコマンド実行者のユーザ名である$USERまたは-uで指定した値，「プロジェクト名」にはdefault または-pで指定した値が代入されます．
dockerコマンドでコンテナ/イメージを削除するには下記コマンドを用いることも可能です．
% docker rmi bonito/ユーザ名/プロジェクト名:latest
% docker rm bonito.ユーザ名.プロジェクト名

また，latestの部分は-tagオプションで指定することも可能です．これにより，過去に作成したイメージにもアクセスできます．
